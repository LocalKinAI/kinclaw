# v1.14.2 — Kit-debt repayment regression suite

v1.14.2 lifted ~400 lines of kit-shape helpers from `pkg/skill/` into
the kinkit repos:

- **kinax-go v0.4.0** got `Element.NavigateMenu(path)` +
  `Element.MenuItemShortcut()` + `ActionScrollToVisible` +
  `AttrMenuItemCmd*` constants.
- **sckit-go v0.3.0** got `DiffImages(a, b, rows, cols)` +
  `DiffGrid.Dirty / BoundingBox / Render`.
- **input-go v0.3.0** got `PasteText(ctx, text, opts...)` +
  `ReadClipboard` / `WriteClipboard`.

The kinclaw skill-layer wrappers (`pkg/skill/ui_extras.go`,
`screen_extras.go`, `input_extras.go`) became thin pass-throughs
to those new kit functions. v1.14.2 ships **byte-equivalent verb
output** vs v1.14.1 — the changes are structural, not behavioral.

This dossier verifies that.

---

## T1 · `ui menu_path` (kinax NavigateMenu)

What changed: `pkg/skill/ui_extras.go::menuPath` used to walk
`AXMenuBar → AXMenuBarItem → AXMenu → AXMenuItem` with inline AXPress
+ 80ms settle. Now it's a one-liner: `app.NavigateMenu(path)`.

```bash
# Setup: open Safari to a normal browsing window.
open -a Safari

# Run:
./kinclaw -exec "ui menu_path 'Safari > File > New Window'" --bundle com.apple.Safari

# Pass: a new Safari window opens. Verb output prints "ok: pressed leaf".
# Fail mode (kit broken): "menu path 'Safari > File > New Window' not found
#         at depth 2" or NavigateMenu returns a not-found error.
```

## T2 · `ui shortcut` (kinax MenuItemShortcut)

What changed: the shortcut readout used to manually pull
`AXMenuItemCmdChar` / `AXMenuItemCmdModifiers` / `AXMenuItemCmdVirtualKey`
attribute by attribute and decode the bits. Now it's
`elem.MenuItemShortcut()` returning typed `(char, mods, vk)`.

```bash
# Run:
./kinclaw -exec "ui shortcut 'Safari > File > New Window'" --bundle com.apple.Safari

# Pass: returns "⌘⇧N" exactly. (NOT "Cmd+Shift+N", NOT "⌘⇧n".)
# Fail mode: empty string, or "(virtual_key=N)" fallback when the
#         menu item has a real char (means MenuItemShortcut returned 0
#         for char even though one exists in AX).
```

## T3 · `screen diff_screenshots` (sckit DiffImages)

What changed: `pkg/skill/screen_extras.go` used to have a
`computeDiffGrid` + `flagDirtyCells` + `renderHeatmap` trio
(~150 lines). Now it's `sckit.DiffImages(...)` + `grid.Dirty(8)` +
`grid.Render(8)`.

```bash
# Setup:
./kinclaw -exec "screen capture --target display --out /tmp/before.png"
# (do something visible — e.g. open a new tab, scroll, anything)
./kinclaw -exec "screen capture --target display --out /tmp/after.png"

# Run:
./kinclaw -exec "screen diff_screenshots /tmp/before.png /tmp/after.png"

# Pass: returns
#       - non-zero "dirty cells" count
#       - bbox in display-local px
#       - ASCII heatmap with '#' marks where you made the change
# Sanity check: same image twice should report 0 dirty cells.
./kinclaw -exec "screen diff_screenshots /tmp/before.png /tmp/before.png"
# ↑ Pass: "no changes / 0 dirty cells".
```

## T4 · `input paste` (input PasteText)

What changed: paste used to be inline `pbcopy → Hotkey(cmd, V) →
sleep → pbcopy(prev)` in `input_extras.go`. Now it's
`input.PasteText(ctx, text, opts...)`.

```bash
# Setup: open TextEdit, click into the document so a text field has focus.
open -a TextEdit -n
# Pre-load clipboard with something distinctive so we can verify restore:
echo "ORIGINAL_CLIPBOARD_$(date +%s)" | pbcopy

# Run:
./kinclaw -exec "input paste '你好世界 1234'"

# Pass:
#   1. "你好世界 1234" appears in TextEdit (no dropped chars — IME-safe)
#   2. After ~200ms, pbpaste returns ORIGINAL_CLIPBOARD_... again
pbpaste
# Fail mode: dropped CJK chars (means PasteText fell through to Type),
#            or clipboard stays as "你好世界 1234" (means restore step broken)
```

## T5 · `ui scroll_to` (kinax ActionScrollToVisible)

What changed: scroll-to-visible used to be a hardcoded
`elem.Perform("AXScrollToVisible")` string. Now it's
`elem.Perform(kinax.ActionScrollToVisible)`. Pure constant lift —
should be byte-equivalent — but verify the constant matches.

```bash
# Setup: open System Settings → Privacy & Security (long scrollable list).
# Pick any element below the fold (e.g. "Lockdown Mode").

./kinclaw -exec "ui scroll_to 'Lockdown Mode' --bundle com.apple.systempreferences"

# Pass: System Settings auto-scrolls to bring the element into view.
# Fail mode: "no action AXScrollToVisible" (means kinax constant
#         is wrong) or no scroll happens (action existed but did
#         nothing — usually means we're calling it on the wrong elem).
```

## What's specifically NOT covered here

- **Browser-tab-like targets**: NavigateMenu walks the system menu bar,
  not in-page menus. Don't try to test it on web app menus.
- **App-not-running case**: NavigateMenu fails-fast if the bundle
  isn't running, which is correct but not tested here.
- **Sandboxed apps that block PID-targeted events**: PasteText with
  `WithPID` may silently fall back; covered in input-go's own test
  suite, not here.
